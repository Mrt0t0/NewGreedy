Overview

NewGreedy is a transparent mitmproxy-based addon that intercepts BitTorrent tracker announces and injects simulated upload statistics. It targets a configurable ratio (default 1.60) using a smart catch-up algorithm with exponential decay, stagnation simulation, and DUL smoothing.

Version 1.6 adds full lifecycle management, dead tracker detection, anti-detection hardening, and a redesigned Web UI with real-time health monitoring.

Architecture

  • newgreedy.py — Entry point. Launches mitmproxy + optional Web UI thread.
  • newgreedy_addon.py — Core mitmproxy addon. Intercepts all /announce requests and patches the uploaded field.
  • newgreedy_web.py — FastAPI server exposing the Web UI and REST/WebSocket API.
  • config.ini — All tunable parameters.
  • stats.json — Persisted per-torrent state (auto-saved every 5 min).

Setup

pip install -r requirements.txt
mitm-install-ca  # install mitmproxy CA certificate
python newgreedy.py

Configure your BitTorrent client (qBittorrent, Transmission…) to use HTTP proxy 127.0.0.1:3456. The Web UI is available at http://localhost:8080.

Docker: docker compose up -d — proxy on port 3456, Web UI on 8080.

Configuration Reference

[proxy]

  • listen_port — mitmproxy listen port (default: 3456)
  • tracker_timeout — seconds before giving up on tracker connection

[spoofing]

  • target_ratio — ratio target to reach (default: 1.60)
  • target_ratio_buffer — safety margin added to target (default: 0.03)
  • catch_up_factor — aggressiveness of upload catch-up (0.1–0.5)
  • max_simulated_speed_mbps — cap on injected upload speed
  • stagnation_probability — probability of simulating a stagnation event
  • upload_noise_pct — random noise % added to uploads for naturalness

[anti_detection]

  • spoof_peer_id — generate a realistic peer_id per torrent (persisted)
  • spoof_user_agent — match User-Agent to the spoofed client
  • dul_smooth_alpha — smoothing factor for +DUL distribution (0=very smooth, 1=instant)
  • tracker_blacklist — comma-separated domains to never inject

[advanced]

  • interval_jitter_pct — ±% randomization of announce interval (default: 0.15 = ±15%)
  • startup_grace_seconds — ignore announces for N seconds after start (default: 15)
  • stats_save_interval — periodic save interval in seconds (default: 300)
  • dead_tracker_threshold — consecutive errors before marking dead (default: 3)
  • dead_tracker_ttl_hours — blacklist duration in hours (default: 12)
  • end_ttl_minutes — minutes without announce before marking torrent as END (default: 60)
  • purge_ttl_minutes — minutes after END before purging from stats (default: 120)

Torrent Lifecycle

Each torrent tracked by NewGreedy goes through the following states:

  • DL+SEED — Active download with upload injection.
  • SEED — Pure seeder (downloaded=0, left=0).
  • CONV — Ratio converging to target.
  • CONV_DONE — Ratio stabilized at target, minimal injection.
  • END — No announce received for end_ttl_minutes. Torrent likely removed.

Torrents in END state are displayed dimmed in the UI and purged from stats.json after purge_ttl_minutes.

Tracker Management

NewGreedy monitors all tracker responses. If a tracker returns ≥ dead_tracker_threshold consecutive errors (HTTP 4xx/5xx or connection timeout), it is flagged as DEAD and blacklisted for dead_tracker_ttl_hours hours. During this period, no DUL is injected and no connection is attempted.

Dead trackers are shown in the Dashboard health bar and alerts section. The blacklist resets automatically after the TTL expires.

Announce timeouts (client disconnects before server responds) are logged as [WARN] CONNECT_ERROR with a running counter visible in the health API.

Anti-Detection Features

  • Announce jitter — Interval randomized ±15% around the tracker-provided interval to avoid machine-regular patterns.
  • DUL smoothing — Upload delta spread over multiple announces using exponential smoothing to avoid abrupt spikes.
  • Persistent peer_id — Each torrent keeps its peer_id stable across restarts (stored in stats.json).
  • Monotone uploaded — uploaded value never decreases between announces (immediate tracker flag).
  • User-Agent preservation — UA is matched to the spoofed client identity and never overwritten with a proxy UA.
  • HTTP tracker support — DUL injection applies equally to HTTP (port 80/1337) and HTTPS trackers.

FAQ

Q: Does NewGreedy work with UDP trackers?
No — mitmproxy only intercepts HTTP/HTTPS. UDP announces pass through unmodified. Disable UDP trackers in your client for full coverage.

Q: Why is a tracker showing as DEAD?
The tracker returned 3+ consecutive errors. Check the Logs page for details. It will auto-recover after 12 hours.

Q: Stats were lost after a restart.
Ensure persist_stats = true in config.ini and that stats.json is writable. The periodic save runs every 5 minutes — a hard kill within that window may lose the last few announces.

Q: A torrent disappeared from the dashboard.
It was purged after being in END state for purge_ttl_minutes. Increase purge_ttl_minutes to retain entries longer.
